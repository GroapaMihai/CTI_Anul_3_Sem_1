Laboratorul 11 - Recapitulare

  • Responsabil: Vladimir Oltean
  • Data publicării: 8.1.2015
  • Data ultimei modificări: 15.01.2016

avrasm.zip schelet_final

Obiective

In acest laborator ne propunem sa definitivam conceptele invatate pe parcursul semestrului, reamintindu-ne etapele prin care am trecut in construirea procesorului nostru.
Recapitularea va fi facuta sub forma unui joc pe echipe. Va exista un numar prestabilit de taskuri (intrebari sau cerinte), iar echipa care va raspunde corect prima va primi punctajul pe acea intrebare. La final, punctajul se va imparti intre echipe, conform numarului de taskuri la care a raspuns in mod corect fiecare.

Taskuri

Acestea vor fi facute publice de asistent in timpul laboratorului, unul cate unul. Daca taskul implica scriere de cod, acesta va fi verificat pe placute sau in simulator de catre toti studentii inainte de trecerea la exercitiul urmator.

Echipa care raspunde la o intrebare este responsabila si cu lamurirea intrebarilor din partea colegilor. Daca echipa nu poate motiva, sau nu poate da un raspuns corect la o intrebare a colegilor/asistentului, atunci va primi punctaj partial pe task.

  • 1. Cate registre are procesorul implementat de noi? Ce dimensiune are fiecare registru?
  • 2. Sunt toate registrele accesibile de catre toate instructiunile? Dati exemplu de instructiune care nu poate accesa toate registrele si explicati de ce.
  • 3. Cum facem daca dorim totusi sa executam un LDI intr-unul din registrele R0-R15?
  • 4. De ce procesorul nostru executa in mod corect instructiuni precum BREQ/BRNE pe care nu le-am implementat in mod explicit?
  • 5. Cum este setat registrul SREG in urma executiei urmatorului cod?
        ldi     R16, 0x80
        ldi     R17, 0x80
        add     R16, R17
  • 6. Dar in urma executiei acestui cod?
        ldi     R16, 0xFF
        ldi     R17, 0x7F
        add     R16, R17
  • 7. Desenati diagrama asociata pentru urmatoarele semnale. Mentionati daca ele vor fi implementate in logica combinationala sau secventiala:
    wire      a, b;
    reg       c, d, e, f;
    reg [1:0] counter;
 
    always @(posedge clk)
        if (reset)
            counter <= 0;
        else
            counter <= counter + 1;
 
    assign a = (counter == 3);
 
    assign b = (counter == 3) ? 1'b1 : 1'b0;
 
    always @*
        if (counter == 3)
            c = 1;
 
    always @*
        if (counter == 3)
            d = 1;
        else
            d = 0;
 
    always @(posedge clk)
        if (counter == 3)
            e <= 1;
 
    always @(posedge clk)
        if (counter == 3)
            f <= 1;
        else
            f <= 0;
  • 8. Ce rol au registrele din spatiul I/O?
  • 9. Dati doua exemple de modalitati diferite prin care poate fi accesat spatiul I/O.
  • 10. Intre ce adrese este mapat in memorie spatiul de registre I/O? Ce altceva mai este mapat in spatiul de adrese si care este prima adresa valida din acea zona?
  • 11. Studiati codul procesorului nostru, in special modulul bus_interface_unit.v. Presupunem urmatoarea secventa de cod:
        ldi     XL, 0x53
        ldi     XH, 0x00
        ld      R16, X

Care va fi adresa care va circula pe bus_addr in timpul instructiunii a treia? De ce?

  • 12. Dupa cum stiti, se spune ca registrele generale sunt o memorie mai rapida decat memoria RAM de date. Insa pentru procesorul nostru, atat registrele, cat si memoria de date sunt implementate in aceeasi tehnologie, si anume SRAM. De ce ar fi, totusi, registrele mai rapide decat memoria de date?
  • 13. Registrul program counter este accesibil din ISA? Se poate scrie in el? Se poate citi din el?
  • 14. Ce fel de endianness are procesorul nostru?
  • 15. Ce surse de ceas are procesorul nostru? Ce sunt trace_mode si trace_clk si cum ar putea fi ele folosite?
  • 16. Ce este un prescaler? Cum functioneaza si la ce il folosim in procesorul nostru?
  • 17. Care este diferenta dintre reset sincron si reset asincron? Ce fel de reset foloseste procesorul nostru si de ce?
  • 18. Care este diferenta dintre o instructiune RETI si una RET? Cati cicli de ceas dureaza fiecare instructiune si de ce? Ce se intampla in fiecare ciclu de ceas?
  • 19. De ce trebuie sa avem multe instructiuni RETI la inceputul codului daca avem intreruperi?
  • 20. Scrieti un cod care afiseaza un text pe display-ul pe 7 segmente folosind intreruperi.
